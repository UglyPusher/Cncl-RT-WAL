Файл taskwrapper.hpp - примерная структура обертки для задачи. Может быть использована, но не является догмой.

Файл tags.hpp - вспомогательные данные для враппера.

Файлы 
    design_sys_registry.md
    sys_registry_structure.md
    working_technology.md
    wrapper_structure.md

содержат описание устройства системного регистра, враппера задачи и технологии работы с ними разработчика.

ЗАДАЧА: Результатом работы будет является документ, описывающий контракты/инрефейсы/структуру каналов связи между задачами.

Все консенсусы фиксируем в файле concensuses.md !Не перезаписывается, только добавляется. клин-ап и пересогласование происходит только по команде оператора

Все проблемы фиксируем в файле problems.md !Содержит только активные проблемы

Все отвергнутые решения фиксируем в файле decline.md !Не перезаписывается, только добавляется. клин-ап и пересогласование происходит только по команде оператора

Все зафиксированные решения фиксируем в файле fixed.md !Не перезаписывается, только добавляется. клин-ап и пересогласование происходит только по команде оператора


На выходе должны быть два файла соответсвенно: chanel_structure.md и channel_technology.md описывающие структуру канал и технологию работы с ним разработчкика (включая взаимосвязи с остальными структурами)

работа ведется в файле structure_thinkings.md Все замечания, предложения, возражения, комментарии дописываются в конец этого файла без перезаписи предыдущего содержимого в формате [сессия НННН][Кодекс/клод/оператор] замечания, предложения, возражения, комментарии.
