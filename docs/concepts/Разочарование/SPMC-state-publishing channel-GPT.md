# SPMC State Publishing Channel — Task Definition

`docs/contracts/SPMC-state-publishing_channel-task.md`
Revision 1.0 — February 2026

---

## 1. Назначение

Определить примитив передачи **состояния (state snapshot)** от **одного writer** к **нескольким readers**.

Семантика:

**latest-wins snapshot channel**

* readers получают согласованный снимок состояния,
* промежуточные публикации могут теряться,
* readers не обязаны получать одинаковые версии.

Примитив предназначен для **RT и SMP систем** с ограниченной памятью.

---

## 2. Модель

### 2.1 Участники

**Writer**

* ровно один поток или ядро
* выполняет `publish(value)`
* может публиковать с произвольной частотой

**Readers**

* N ≥ 1 независимых потоков или ядер
* выполняют `read(out)`
* могут выполняться параллельно

---

## 3. Операции

### 3.1 Publish

```
publish(T value)
```

Writer публикует новое состояние.

Требования:

* операция завершается за конечное число шагов
* операция не блокируется readers

---

### 3.2 Read

```
read(T& out)
```

Reader получает snapshot состояния.

Требования:

* операция завершается за конечное число шагов
* операция не блокируется writer
* операция всегда возвращает значение

---

## 4. Семантика

### 4.1 Latest-wins

Если writer публикует последовательность:

```
S0 → S1 → S2 → S3
```

Reader может получить:

```
S0
S2
S3
```

Пропуски допустимы.

---

### 4.2 Snapshot

Read возвращает состояние, которое соответствует **одной завершённой публикации**.

Недопустимо:

```
часть S1 + часть S2
```

---

## 5. Гарантии корректности

### G1. No Torn Read

Невозможно состояние:

```
reader читает slots[i]
writer пишет slots[i]
```

в перекрывающиеся интервалы времени.

---

### G2. Publication Consistency

Если reader получил состояние S, то:

```
S = value переданный publish()
```

для некоторого завершённого publish.

---

### G3. Single Writer Coherence

Публикации writer наблюдаются readers в некотором порядке writer.

Readers могут пропускать публикации.

---

## 6. Гарантии прогресса

### P1. Wait-Free Read

Для любого reader:

```
read()
```

завершается за ограниченное число шагов независимо от поведения writer.

---

### P2. Wait-Free Publish

```
publish()
```

завершается за ограниченное число шагов независимо от поведения readers.

---

## 7. Ограничения памяти

Память ограничена фиксированным числом слотов:

```
K slots
```

где:

```
K = f(N)
```

и N — максимальное число readers.

---

## 8. Ограничения модели

### 8.1 SMP

Writer и readers могут выполняться одновременно на разных ядрах.

---

### 8.2 Preemption

Readers могут быть вытеснены:

* между любыми инструкциями
* на произвольное время

Writer может быть вытеснен:

* между любыми инструкциями
* на произвольное время

---

### 8.3 Memory Model

Атомики соответствуют C++ memory model.

Допускается:

```
acquire / release
```

Гарантируется:

* atomic word access
* no torn atomics

---

## 9. Минимальные требования

### R1

Readers могут одновременно читать один и тот же snapshot.

---

### R2

Reader удерживает не более одного snapshot одновременно.

---

### R3

Writer не зависит от scheduler fairness.

---

## 10. Запрещённые предположения

Следующие предположения **не допускаются**:

* disable preemption требуется для корректности
* bounded latency readers требуется для корректности
* bounded publish rate требуется для корректности
* scheduler fairness требуется для корректности

---

## 11. Формулировка задачи

Найти реализацию канала состояния SPMC, такую что:

* выполняются G1–G3
* выполняются P1–P2
* выполняются ограничения модели
* память ограничена K = O(N)

---

## 12. Известная сложность задачи

Одновременное выполнение:

* SMP
* fully preemptible readers
* wait-free read
* wait-free publish
* no torn read
* bounded memory

создаёт сильные ограничения на возможные реализации.

Задача состоит в поиске реализации, удовлетворяющей всем условиям.
