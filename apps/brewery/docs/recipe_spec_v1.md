# Спецификация рецепта v1 (brewery)

Документ описывает формат “рецепта” для режима **АВТО** в v1. Рецепт не содержит safety‑порогов и не может отменять interlock’и.

## Термины

- **Рецепт** — набор параметров процесса варки (профиль температур/таймеры/режим помпы), исполняемый FSM режима АВТО.
- **Системные константы** — параметры safety/алгоритмов (пороги, гистерезисы, тайм‑ауты), редактируются в ИНИТ как “System”, не часть рецепта.
- **Шаг** — элемент последовательности (например термопауза).

## Scope v1

### Поддерживаемые сущности

- Профиль затирания: список термопауз `{target_temp_c, hold_min}`.
- Варка (кипячение): длительность `{boil_min}`.
- Сигналы внесения хмеля/добавок: список таймеров `{minutes_before_end}`.
- Охлаждение: целевая температура `{chill_target_temp_x2}` (только оповещение, без управления).
- Помпа рециркуляции во время затирания:
  - автоматическое отключение при `temp >= 80°C` (правило safety/интерлок системы);
  - опционально цикл “работа/пауза” (параметр рецепта).

### Не входит в рецепт v1

- Любые safety‑пороги (температурные лимиты, тайм‑ауты аварии, гистерезисы interlock’ов).
- Параметры ПИД (коэффициенты/ограничения).
- Параметры железа/калибровок.
- Автоматизация “что делать после варки” (ферментация, CIP).

## Ограничение по размеру (цель v1)

В идеале один рецепт (одна запись в Flash) должен укладываться в **512 байт** целиком (заголовок + payload).

Следствия для v1:

- держать строки короткими (имя 20 символов достаточно для LCD2004);
- ограничить максимальное число шагов/событий;
- по возможности избегать “свободных строк” внутри рецепта (помимо `name`), если нет явной необходимости.

## Модель данных (логическая)

Ниже логическая модель (в терминах структур/полей). Хранилище может быть бинарным, но поля и их смысл фиксируются здесь.

### RecipeV1

- `name` (string): имя рецепта для UI.
- `batch_volume_l` (u16): объём партии (информативно, не участвует в safety v1).
- `mash_steps[]` (array of MashStep): термопаузы затирания.
- `recirc` (Recirculation): параметры помпы рециркуляции в затирании.
- `boil_min` (u8): длительность кипячения (0..255 минут).
- `boil_additions[]` (array of BoilAddition): оповещения (сигнал).
- `chill_target_temp_x2` (u16): целевая температура окончания охлаждения, шаг 0.5 °C (сигнал).
- `flags` (u32 bitset): зарезервировано под расширения (v1 все биты = 0).

### MashStep

- `target_temp_x2` (u16): целевая температура *в половинках градуса* (например 129 = 64.5 °C).
- `hold_min` (u8): длительность удержания на паузе (0..255 минут).

### Recirculation

- `enabled` (bool): включает режим управления рециркуляцией в АВТО.
- `duty_on_s` (u8): сколько секунд работать (0..255).
- `duty_off_s` (u8): сколько секунд отдыхать (0..30 в v1).
- `pump_id` (u8): какая помпа считается “рециркуляцией” (v1: 0/1).

Примечания:

- Interlock “выключить рециркуляцию при 80°C” — не часть рецепта. Рецепт лишь задаёт желаемый режим, а interlock может его отменить.

### BoilAddition

- `minutes_before_end` (u8): сколько минут до конца кипа подать сигнал (0..255).
Примечание v1:

- `label` не используется: технолог ориентируется по таймеру (в минутах до конца кипа).

## Валидация (обязательная)

Валидация выполняется:

- при сохранении рецепта в ИНИТ;
- при загрузке рецепта в АВТО;
- при чтении из Flash (CRC/формат).

### Общие правила

- Нельзя сохранять рецепт, который не проходит валидацию.
- Нельзя стартовать АВТО с невалидным рецептом (ошибка + лог).
- Рецепт может быть “валидным”, но процесс всё равно может быть остановлен safety‑логикой (это ожидаемо).

### Лимиты v1 (рекомендуемые значения)

- `name`: 1..20 ASCII/UTF‑8 символов (UI LCD2004).
- `batch_volume_l`: 1..2000 (информативно).
- `mash_steps`: 1..10 шагов.
- `target_temp_x2`: 0..200 (0..100.0°C), но для затирания рекомендовать 60..160.
- `hold_min`: 0..255.
- `boil_min`: 0..255.
- `boil_additions`: 0..10 событий.
- `minutes_before_end`: 0..`boil_min`.
- `chill_target_temp_x2`: 0..200 (0..100.0°C).
- `recirc.duty_on_s`: 0..255.
- `recirc.duty_off_s`: 0..30.
- `recirc`: если `enabled==true`, то `duty_on_s + duty_off_s > 0`.

### Семантические правила

- `mash_steps` — исполняются строго по порядку.
- Дубли температур допускаются (например две паузы на одной температуре) — это валидно.
- Если `boil_min==0`, то `boil_additions` должен быть пуст (иначе ошибка).

## Редактирование в UI (ИНИТ) — сценарии v1

Минимум:

- Список рецептов: создать/удалить/выбрать.
- Экран рецепта: имя, объём, список шагов, длительность кипа, события кипа, целевая t охлаждения, параметры рециркуляции.
- Дерево параметров: навигация Up/Down, вход/выход Enter/Exit, изменение значения Up/Down.

Поведение:

- “Сохранить” доступно только если рецепт валиден.
- “Старт АВТО” доступен только если выбранный рецепт валиден.

## Хранение (Flash) и версионирование (proposal v1)

Предлагаемый формат: “запись с заголовком + payload + CRC”.

### Заголовок

- `magic` (u32): `'BREW'`
- `version` (u16): версия формата в виде `MMmm` (major/minor), где:
  - сотни/тысячи/десятки тысяч = **major**;
  - единицы/десятки = **minor**.
  - пример: `102` = версия **1.02**.
- `payload_len` (u16)
- `payload_crc32` (u32): CRC32 от payload

### Payload

- Бинарная сериализация фиксированных полей + массивов фикс. длины с `count`.
- Строки: фиксированный массив байт + `len` (без нулевого терминатора).

Рекомендация v1 по укладыванию в 512 байт:

- Держать payload фиксированного максимального размера (например 128..256 байт), чтобы упрощать запись/чтение и оставлять запас под расширения.
- В v1 хранить только `minutes_before_end` для `boil_additions` (без `label`).

Правила:

- Если CRC/формат неверен — рецепт считается отсутствующим/повреждённым.
- При изменении структуры — увеличиваем `version`, добавляем миграцию или сбрасываем на дефолт.

## Открытые вопросы (recipe)

- Где хранить `batch_volume_l`: строго в рецепте (инфо) или вынести в “System/Batch” (если влияет на расчёты в будущем).
- Хотим ли добавить “человеческое отображение” 0.5 °C на LCD (округление/формат), или достаточно целых градусов в UI.

## Примеры (логические)

### “Single infusion”

- `mash_steps`: `[(65.0°C, 60m), (78.0°C, 10m)]`  (в хранении `target_temp_x2`: 130 и 156)
- `recirc`: `enabled=true, on=30s, off=30s, pump_id=0`
- `boil_min`: `60`
- `boil_additions`: `[60, 15, 5]`
- `chill_target_temp_x2`: `50` (25.0 °C)
